# v0.19.0 Release Notes

**Release Date:** April 11, 2026

## Highlights

This release introduces **security notification letter generation**, enabling teams to create professional SLA exception notifications and security hardening notices directly from the command line or programmatically.

## New Features

### Security Notification Letters

The new `letter` package and `govex letter` CLI command support two types of security notifications:

#### SLA Exception Notifications

When vulnerability remediation extends beyond SLA targets, generate formal notifications that include:

- Finding details with severity and CVSS scoring
- Original and revised SLA timelines
- Reasons for delay
- Risk assessment and mitigating controls
- Remediation plan with milestones
- Approver sign-off

#### Security Hardening Notices

For defense-in-depth improvements that don't represent exploitable vulnerabilities:

- Rationale for the improvement
- Target completion date
- Implementation milestones
- Customer impact assessment

### CLI Commands

```bash
# Generate SLA exception letter from JSON
govex letter generate --input finding.json --output-md letter.md

# Generate hardening notice
govex letter generate --type hardening --input notice.json --output-md letter.md

# Output JSON Schema for AI agent integration
govex letter schema

# Output example templates
govex letter example              # SLA exception
govex letter example --type hardening
```

### Pandoc-Compatible Output

Letter output is Pandoc-compatible Markdown, enabling conversion to professional formats:

```bash
# Convert to DOCX
pandoc letter.md -o letter.docx

# Convert to PDF
pandoc letter.md -o letter.pdf
```

### JSON Schema for AI Integration

The `govex letter schema` command outputs JSON Schema, enabling:

- AI agents to generate valid letter JSON
- IDE autocompletion for letter files
- Validation of letter input files

## Example: SLA Exception Letter

```bash
govex letter generate \
  --type sla \
  --customer "Widget Inc" \
  --application "Payments API" \
  --finding-title "Missing rate limiting" \
  --finding-severity Low \
  --finding-id APPSEC-1234 \
  --finding-date 2026-04-01 \
  --cvss-score 3.1 \
  --cvss-version 4.0 \
  --sla-target-days 90 \
  --sla-original-date 2026-06-30 \
  --sla-new-date 2026-07-31 \
  --delay-reason "Dependent on upstream API gateway change" \
  --risk "Low likelihood of exploitation due to internal-only access" \
  --mitigation "WAF rate limiting rules in place" \
  --mitigation "Monitoring enabled" \
  --remediation "Implement native rate limiting in service layer" \
  --milestone "1:Deploy rate limiting in report-only mode:2026-08-01:None — monitoring only" \
  --milestone "2:Enable rate limiting enforcement:2026-09-01:None expected" \
  --approver-name "John Doe" \
  --approver-title "VP of Engineering" \
  --approver-date 2026-04-10 \
  --sender-name "Jane Smith" \
  --sender-title "Security Engineer" \
  --sender-team "Application Security" \
  --sender-company "Acme Corp" \
  --sender-email "security@acme.com" \
  --output-md letter.md
```
